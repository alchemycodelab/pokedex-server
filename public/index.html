Pokedex API

API endpoints

Get Pokedex

    
    https://alchemy-pokedex.herokuapp.com/api/pokedex

Get Pokemon

Note that this uses the _id property value, not the id property. 

    
    https://alchemy-pokedex.herokuapp.com/api/pokedex/5cef3501ef6005a77cd4fd33

Get Types, Abilities, Shapes, or Egg Groups

Returns distinct values list with count (see data format below)

    
    https://alchemy-pokedex.herokuapp.com/api/pokedex/types

    https://alchemy-pokedex.herokuapp.com/api/pokedex/abilities

    https://alchemy-pokedex.herokuapp.com/api/pokedex/shapes

    https://alchemy-pokedex.herokuapp.com/api/pokedex/eggGroups

Options

Paging

You can specify page and perPage via querystring. Defaults to first page and 20 pokemon per page. 

    
    https://alchemy-pokedex.herokuapp.com/api/pokedex?page=3&perPage=50

Searching

You can specify any object property as a query. Strings match if they are included in the specified property, numbers properties match if they are greater than or equal to the supplied value.

If you specify type, ability, shape, or eggGroup, it will match across all the related fields, for example type will match type_1 or type_2. 

    
    https://alchemy-pokedex.herokuapp.com/api/pokedex?pokemon=char&defense=50

Sorting

Specify a sort property as a query parameter. Optionally specify a direction query parameter with either "asc" or "desc", defaults to ascending. 

    
    https://alchemy-pokedex.herokuapp.com/api/pokedex?sort=defence&direction=desc

Data Format

Pokedex return format is as follows: 

    
    {
        "count": 6,
        "page": 1,
        "perPage": 20,
        "sort": {
            "by": "pokemon",
            "direction": "asc"
        },
        "search": {
            "pokemon": "char"
        },
        "results": [
            // pokemon objects (see below)
        ]
    }
